# HTML Output Generator

会話内容や markdown ファイルを、見やすい HTML ファイルに変換するスキル。HTML の組み立て本体は**トークン消費の少ない sonnet サブエージェント (`html-builder`) に必ず委譲**する。

## 基本方針

- **このスキル自身は HTML を組み立てない**。必ず `Task` ツールで `subagent_type: "html-builder"` を起動して委譲する
- **受け取った情報以上の情報を追加しない**。要約・解釈・例示・補足の追加は禁止 (skill 側でも agent 側でも禁止)
- UI 規約: シンプル / 不要な色付け禁止 / 重要箇所と比較のみ色分け / 図はフロー・アーキテクチャ説明がある場合のみ mermaid 挿入

## 実行手順

### Step 1: 入力ソースの決定 (引数 > 直近の会話の文脈)

ユーザー引数 `$ARGUMENTS` を確認する:

1. **引数あり**: `$ARGUMENTS` をファイルパスとして扱う
   - `Bash` で `test -f "$ARGUMENTS" && echo OK` 相当の存在確認をする
   - 存在すればそのパスを絶対パスに正規化し `INPUT_PATH` とする
   - 存在しなければ「指定ファイルが見つかりません」とユーザーに通知して終了
2. **引数なし**: 直近の会話の文脈から HTML 化対象を組み立てる
   - 会話履歴で**直前に assistant が出力した markdown / 主要テキスト**を抽出する (例: 直前のターンで生成した手順書 / アーキテクチャ説明 / レビュー結果など)
   - 抽出した内容を `Write` で `/tmp/html-output-input-<timestamp>.md` に書き出す (timestamp は `Bash` の `date +%Y%m%d-%H%M%S` で取得)
   - そのパスを `INPUT_PATH` とする
   - **抽出時の注意**: 会話で実際に出力された内容のみを書き出す。要約や補足を加えない

### Step 2: 出力ファイル名の決定 (会話トピックから自動命名)

入力テキストから出力ファイル名 (slug) を決定する:

1. 入力の H1 (`# ...`) があればそれをスラグ化に使う
2. H1 がなければ最初の見出し (H2 等) または冒頭の主要キーワードを使う
3. スラグ化規則:
   - 英字は小文字化
   - 日本語などの非 ASCII は `Bash` で簡易ローマ字化が困難なため、ASCII 部分のみを抽出して使う
   - ASCII 部分がない場合は固定文字列 `output` を使う
   - 連続するスペース / 記号 / 句読点は `-` に変換
   - 連続する `-` は 1 つに圧縮、前後の `-` は除去
   - 最大 50 文字に切り詰め
4. 拡張子は `.html`
5. **衝突回避**: 現在のワーキングディレクトリに同名ファイルがあれば `-2`, `-3` の suffix を付ける (`Bash` で `test -f` を順次確認)

最終的な絶対パスを `OUTPUT_PATH` とする。

### Step 3: サブエージェントへ委譲

`Task` ツールを以下のパラメータで起動する:

- `subagent_type`: `"html-builder"`
- `description`: `"markdown を HTML へ変換"` (3〜5語)
- `prompt`: 以下の情報を含むテキスト
  - `INPUT_PATH`: Step 1 で確定した入力ファイルの絶対パス
  - `OUTPUT_PATH`: Step 2 で確定した出力 HTML の絶対パス
  - `TEMPLATE_PATH`: `${CLAUDE_PLUGIN_ROOT}/skills/html-output/assets/template.html` の絶対パス
  - **再度強調する制約** (agent 側でも明記済みだが skill 側でも prompt に含める):
    - 入力にない情報の追加・要約・解釈は禁止
    - 装飾色は `.highlight` / `mark` / `.diff-old` / `.diff-new` のみ
    - mermaid は入力にフロー/アーキテクチャ/シーケンスの記述がある場合のみ挿入
    - 完了後は出力ファイル絶対パスのみを 1 行で返すこと

### Step 4: 完了報告

サブエージェントから返ってきた出力パスをそのままユーザーに報告する。

報告フォーマット例:

```
HTMLを生成しました: /path/to/output.html
```

一時 markdown (`/tmp/html-output-input-*.md`) を作成した場合はそのまま残す (再実行 / デバッグの容易性のため明示削除はしない)。

## 禁止事項 (再掲)

- skill 自身で HTML を組み立てること (必ず `html-builder` に委譲する)
- 入力にない情報の追加・要約・解釈・例示
- 装飾色を `.highlight` / `mark` / `.diff-old` / `.diff-new` 以外で使うこと
- 入力に図的記述がない場合に mermaid を追加すること

## エラーハンドリング

| ケース                                 | 対処                                                                                     |
| -------------------------------------- | ---------------------------------------------------------------------------------------- |
| 引数のファイルが存在しない             | ユーザーに通知して終了                                                                   |
| 引数なしで直前会話に HTML 化対象が無い | 「変換対象が特定できません。markdownファイルパスを引数で指定してください」と通知して終了 |
| `html-builder` 起動に失敗              | エラー内容をそのまま報告                                                                 |
| 出力パスへの書き込みに失敗             | エージェントからのエラーを転送                                                           |

## 注意事項

- `${CLAUDE_PLUGIN_ROOT}` はプラグインのルートディレクトリに展開される環境変数
- 出力先は常に**現在のワーキングディレクトリ**。グローバル位置 (例: `/tmp`) には書かない (一時 markdown を除く)
- `html-builder` エージェントは sonnet モデルで動く。Opus を浪費しないことが本プラグインの主目的
